## 2. 比較結果

| 指標 | 候補A・交差流 | 候補B・段流路 | 候補C・開港型 |
|---|---:|---:|---:|
| ガラス反射経路 | 359 | 146 | 330 |
| 2回反射経路 | 652 | 546 | 516 |
| 共通3タップ参考得点 | 2747 | 2192 | 2650 |
| ビーコン同士の最小間隔 | 44.18px | 50.12px | 118.12px |
| ビーコンとガラスの近接警告 | 2件 | 1件 | 1件 |
| ガラス端点の画面端からの最小余白 | 60px | 54px | 42px |

この数値は、同じ121地点、同じ56到達時刻、同じ3タップ地点で調べた結果です。実際の面白さや見やすさを直接測った値ではありません。

## 3. 候補Cを実装する理由

候補Cは、ビーコン同士の最小間隔が118.12pxで3候補中最も広く、3つを見分けやすい可能性があります。

ガラス反射経路は330あり、候補Aの359に近い数を保っています。共通3タップ参考得点も2650で、候補Aの2747に近い水準です。

近接警告は1件で、候補Aの2件より少なくなっています。

## 4. 残る注意点

ガラス端点の画面端からの最小余白は42pxで、3候補中最も小さい値です。iPhone SE級で端のガラス片を見分けられるかは、実機確認が必要です。

候補Cで実装を進めますが、実機で重大な見づらさが確認された場合は、ガラス片を内側へ寄せた候補Dを作り、同じ分析条件で再比較します。

## 5. Phase 3Cで行うこと

- 候補Cの座標、速度、配置ID、指紋、ルール版を本番設定へ固定する。
- 公式モードの初期化では`Math.random()`を使わない。
- 従来のランダム配置を練習モードへ残す。
- 練習結果からランキング送信を呼べないようにする。
- 旧ランダム配置の記録と混ざるため、公式モードの実送信もPhase 5まで停止する。

## 6. 公開前に確認すること

- iPhone SE級で候補Cのガラス片を識別できるか。
- WebGLとCanvas 2Dで同じ配置に見えるか。
- 3タップの選択肢が一つに固定されすぎないか。
- Phase 4の反射・得点実装後も候補Cが適切か。

候補Cの選定は実装開始の決定です。公開版としての最終承認は、実機確認と後続Phaseの完了後に行います。
