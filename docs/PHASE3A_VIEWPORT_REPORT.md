## 1. 目的

端末の画面サイズを、波の距離、速度、反射時刻、得点へ直接使わないようにする。

ゲーム内部は常に次の大きさで計算する。

```text
論理幅: 360
論理高: 640
表示倍率: min(viewWidth / 360, viewHeight / 640)
```

端末画面は表示先としてだけ使い、物理座標へは使わない。

## 2. 実装

### 共通変換

`src/game/viewport.js`を追加した。

- 画面サイズから表示倍率、表示幅、表示高、左右余白、上下余白を計算する。
- 画面上のPointer座標を360×640の論理座標へ変換する。
- 論理領域外と余白上の入力を拒否する。
- 自動試験用に論理座標から画面座標へ戻す関数を提供する。

### World

- `World.w`を360へ固定した。
- `World.h`を640へ固定した。
- `World.reset()`から画面幅・画面高の引数を削除した。
- 画面サイズ変更ではWorldを作り直さない。

### 入力

- `pointerdown`の`clientX / clientY`を直接`World.tap()`へ渡さない。
- Canvasの画面上の位置、余白、表示倍率を使って論理座標へ変換する。
- 変換に失敗した入力はタップ数へ加えない。
- `PLAYING`中の主入力だけを受ける。
- `pointercancel`を処理する。

### 描画

WebGLとCanvas 2Dの両方へ同じ表示変換を適用した。

- Canvas 2Dは余白を含む画面全体へ暗い水面背景を描き、ゲーム領域を切り取らずに中央へ配置する。
- WebGLは画面全体の背景と、360×640のゲーム領域を別のviewportで描く。
- Device Pixel RatioはCanvasの描画解像度だけへ使う。
- 画面回転とresizeでは表示変換だけを更新し、World状態は変更しない。

## 4. 自動試験

`tests/viewport.test.mjs`へ5件の試験を追加した。

対象画面サイズ:

```text
320×568
375×812
390×844
1024×1366
```

確認項目:

1. 360×640全体が切り取られずに収まる。
2. 左上、中央、右下の往復変換誤差が0.25論理ピクセル以内である。
3. 縦長画面の上下余白と横長画面の左右余白を拒否する。
4. viewportを作り直してもWorld状態が変化しない。
5. 同じ乱数列と同じ論理タップなら、画面サイズが違っても同じWorld状態と得点になる。
