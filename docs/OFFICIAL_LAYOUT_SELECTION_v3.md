## 1. 選定判断

プロジェクト所有者が「候補Cでまずは実装」と明示したため、候補C・開港型を公式配置の実装対象として選定する。

この判断により、Phase 3Bまでの`human-decision-pending`は終了する。比較用の候補A・B・Cデータと比較画像は判断履歴として残す。

## 3. 選定理由

確認済みの比較値では、候補Cは次の性質を持つ。

- ビーコン同士の最小間隔が118.12論理pxで、3候補中で最も広い。
- ガラス反射経路が330あり、候補Aの359に近い。
- 共通3タップ参考得点が2650で、候補Aの2747に近い。
- ビーコンとガラスの近接警告が1件で、候補Aの2件より少ない。

これらは候補Cを実装対象へ選ぶ根拠であり、公開品質の最終合格を意味しない。

## 4. 実装契約

公式モードは候補Cを使い、初期化時に`Math.random()`を使わない。配置ID、指紋、ルール版をWorldへ保持する。

練習モードは従来のランダム配置を残し、Worldへ注入した乱数を使う。練習結果はランキング送信対象にしない。

ランキングの正式な送信契約と旧スコア分離はPhase 5で確定する。このPhaseでは旧ランダム配置の記録と新しい公式配置の記録を混ぜないため、公式モードも実送信を行わない。

## 5. 未確認事項

- iPhone SE級で端側のガラス片を見分けられるか。
- WebGLでビーコンとガラスが重なった時に理解できるか。
- Canvas 2Dでも同じ配置を識別できるか。
- 実際の3タップで候補Cが単調にならないか。
- Phase 4の反射・得点変更後も候補Cが適切か。

候補Cで実装を進めるが、実機確認で重大な問題が出た場合は、候補Cのガラス片を内側へ寄せた候補Dを作り、同じ比較条件へ戻す。

## 6. 戻し方

この選定を取り消す場合は、候補Cの選定記録とPhase 3Cの実装Pull Requestをrevertする。Supabaseへ新しい公式スコアを送らないため、ランキングデータの戻し作業は不要。
